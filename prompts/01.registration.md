Execute the following implementation steps for user registration and email verification. Follow file layouts strictly.

### 1. Security & Helpers Layout
* **src/lib/hash.ts**: Export `hashPassword(password: string)` using `bcrypt` with a salt round of 10.
* **src/lib/email.ts**: Verify existence of SMTP envs (`SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `EMAIL_FROM`). Export a `sendMail(to, subject, html)` function utilizing `nodemailer` (`secure: false`).
* **src/lib/url.ts**: Export `getAppUrl()` returning `http://localhost:${process.env.PORT || 5000}`.

### 2. Validation Schema
* **src/routes/auth.schema.ts**: Create a Zod `registrationSchema` object:
  * `name`: string, min 5 characters.
  * `email`: string, valid email format.
  * `password`: string, min 6 characters.

### 3. Controller Logic (src/controller/auth.controller.ts)

#### registrationHandler:
1. Parse `req.body` with `registrationSchema.safeParse()`. If invalid, return 400 with errors.
2. Normalize `email` to lowercase.
3. Query database for `existingUser` by email. If found, return 400 error.
4. Hash password using `hashPassword`.
5. Generate a temporary verification JWT containing the user's email signed with `JWT_SECRET` (set short expiration).
6. Save user to database (`name`, `email`, `passwordHash`, defaults).
7. Construct confirmation link: `${getAppUrl()}/api/auth/verify-email?token=${token}`.
8. Dispatch verification link via `sendMail` to Mailtrap.
9. Return 201 with `_id`, `email`, `role`, and `isEmailVerified`.

#### emailHandler:
1. Extract `token` from `req.query.token`. If missing, return 400.
2. Verify token payload using `JWT_SECRET`. If invalid/expired, return 401.
3. Extract email from payload, locate user in database.
4. Update `isEmailVerified` to `true`, save document.
5. Return 200 success response.

### 4. Routing (src/routes/auth.routes.ts)
* `POST /registration` -> `registrationHandler`
* `GET /verify-email` -> `emailHandler`

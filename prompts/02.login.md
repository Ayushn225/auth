Execute the implementation steps for user login. Follow file layouts and security rules strictly.

### 1. Code Base Utilities
* **src/lib/hash.ts**: Export `comparePassword(plain: string, hashed: string): Promise<boolean>` utilizing `bcrypt.compare`.
* **src/lib/token.ts**: 
  * Export `createAccessToken(payload: { userId: string, role: string, tokenVersion: number })` -> Expire: 30m.
  * Export `createRefreshToken(payload: { userId: string, tokenVersion: number })` -> Expire: 7d.

### 2. Validation Schema
* **src/routes/auth.schema.ts**: Create a Zod `loginSchema` validating `email` (string, email format) and `password` (string).

### 3. Controller Logic (src/controller/auth.controller.ts)

#### loginHandler:
1. Parse `req.body` with `loginSchema.safeParse()`. If invalid, return 400 with flattened errors.
2. Query database for user by email. If not found, return 401 "Invalid credentials".
3. Check password validity via `comparePassword`. If false, return 401 "Invalid credentials".
4. Check `user.isEmailVerified`. If false, return 403 "Please verify your email".
5. Call `createAccessToken` and `createRefreshToken` passing matching payload criteria.
6. Configure environmental flag: `const isProd = process.env.NODE_ENV === 'production';`.
7. Issue cookie: name `'refreshToken'`, value `refreshToken`, configurations: `{ httpOnly: true, secure: isProd, sameSite: 'lax', maxAge: 7 * 24 * 60 * 60 * 1000 }`.
8. Return 200 with message "Login successful", accessToken, and user fields (`_id`, `isEmailVerified`, `twoFactorEnabled`).

### 4. Routing (src/routes/auth.routes.ts)
* `POST /login` -> `loginHandler`
